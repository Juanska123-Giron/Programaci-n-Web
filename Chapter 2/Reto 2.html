SERVIDORES WEB Y DE APLICACIONES

  • Inicio

Es sustancial comenzar definiendo qué es un servidor, ya que es a partir de los tipos, que se ramifican tantos los servidores web, como los de aplicaciones, de correo, de archivos y demás.

Los servidores son un sistema físico o virtual cuyo fin es proporcionar información, recursos, datos, y servicios a diferentes demandantes enlazados a través de una red pública o privada. Estos funcionan mediante la arquitectura cliente/servidor, sin embargo, cualquier dispositivo u ordenador puede actuar en ambos extremos, pues cada sistema está en la capacidad para distribuir recursos o usar los de otros.

Existen varios tipos, tales como los servidores proxy, de archivos, de correo, FTP, DNS, de aplicaciones, y los web, una lista que abarcaría mayor extensión, más los mencionados anteriormente son los más comunes. Por otro lado, se hará énfasis en los dos últimos, resaltando los servidores web y de aplicaciones, listando unos ejemplos y presentando sus características:

  • Servidores Web: Se basan en el almacenamiento de documentos HTML, imágenes, textos, vídeos, audios, software, y demás conformantes del contenido web, aquel que es distribuido a los clientes que lo solicitan dentro de la misma red haciendo uso de los protocolos de trasferencia de datos como HTTP y HTTPS. Entre los más conocidos están:
    • Apache web server, de código abierto y software gratuito. Es multiplataforma.
    • Nginx, con versión comercial y libre, disponible en Windows, Linux y Unix.
    • LiteSpeed, con diferentes licencias, es una alternativa de Apache para entornos Linux.
    • Microsoft Internet Information Services (IIS), propio para Windows, permite la interpretación de páginas en PHP o Perl.
    • Lighttpd, creado en C y con licencia BSD, tiene la capacidad de servir peticiones de páginas realizadas en cualquier lenguaje de programación.
    • Caddy, servidor escrito en Go, similar a Apache, con la particularidad de validar los sitios con HTTPS.
    • Google Web Server, no disponible al público, es de tipo privado y escrito en C++, pues fundamenta la infraestructura web de la compañía.
    • Cherokee, web server multiplataforma de tipo open source que soporta integración de tecnologías como PHP, FastCGI,SCGI, y demás.

    Es así como la lista de ejemplos sigue, productos como NodeJS, Sun Java System Web Server, y entre otros, son evidencia de esto.

  • Servidores de Aplicaciones: Permiten la ejecución de aplicaciones de forma remota y casi portable, es decir, sin la necesidad de que los clientes lo hagan de manera local. A menudo son los intermediarios que contienen las aplicaciones web o programas que se puedan abrir en un sitio de la red para que cualquier usuario de la plataforma pueda acceder a su ejecución sin tener que descargarlas e instalarlas en sus ordenadores. Como ejemplos se encuentran:
    • Oracle WebLogic, multiplataforma y programado en Java.
    • WebSphere Application Server, multiplataforma de tipo privado.
    • Netscape Application Server.
    • Sun One.
    • Oracle IAS, basado en Java EE.
    • Borland AppServer.
    • HP Bluestone.
